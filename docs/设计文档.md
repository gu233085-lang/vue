# 超市管理系统 设计文档

## 一、项目概述

超市管理系统是一个面向中小型超市的进销存管理平台，支持商品管理、销售记录、进货管理、员工考勤等功能，采用前后端分离架构，数据持久化到 MySQL 数据库。

| 项目 | 说明 |
|------|------|
| 前端框架 | Vue 3 + Vite + Pinia + Vue Router |
| UI 组件库 | Element Plus |
| 图表库 | ECharts |
| 后端框架 | Express (Node.js) |
| 数据库 | MySQL 8.4 |
| 认证方式 | JWT + bcrypt |
| 邮件服务 | Nodemailer (QQ邮箱 SMTP) |

## 三、数据库设计

### 3.1 数据库名

`supermarket`，字符集 utf8mb4。

### 3.2 表结构

#### users（用户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| username | VARCHAR(50) | 用户名，唯一 |
| password | VARCHAR(255) | bcrypt 加密密码 |
| email | VARCHAR(100) | 邮箱 |
| role | VARCHAR(20) | 角色（admin/user） |

#### products（商品表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| name | VARCHAR(100) | 商品名称 |
| category | VARCHAR(50) | 分类（水果/蔬菜/肉类/生活用品/鱼类/服装） |
| sell_price | DECIMAL(10,2) | 售价 |
| cost_price | DECIMAL(10,2) | 进价 |
| stock | INT | 库存数量 |
| unit | VARCHAR(20) | 单位（斤/瓶/件等） |
| threshold | INT | 库存预警阈值 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间（自动更新） |

#### sales（销售记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| product_id | INT | 商品ID（外键） |
| product_name | VARCHAR(100) | 商品名称快照 |
| quantity | INT | 销售数量 |
| unit_price | DECIMAL(10,2) | 成交单价 |
| total_amount | DECIMAL(10,2) | 成交金额 |
| cashier | VARCHAR(50) | 收银员 |
| created_at | DATETIME | 销售时间 |

#### purchases（进货记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| product_id | INT | 商品ID（外键） |
| product_name | VARCHAR(100) | 商品名称快照 |
| quantity | INT | 进货数量 |
| cost_price | DECIMAL(10,2) | 进货单价 |
| total_cost | DECIMAL(10,2) | 进货总金额 |
| supplier | VARCHAR(100) | 供应商 |
| created_at | DATETIME | 进货时间 |

#### employees（员工表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| name | VARCHAR(50) | 姓名 |
| employee_no | VARCHAR(20) | 工号（唯一） |
| position | VARCHAR(50) | 职位 |
| phone | VARCHAR(20) | 电话 |
| hire_date | DATE | 入职日期 |
| created_at | DATETIME | 创建时间 |

#### attendance（考勤表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| employee_id | INT | 员工ID（外键，级联删除） |
| date | DATE | 日期 |
| clock_in | TIME | 上班打卡时间 |
| clock_out | TIME | 下班签退时间 |

> 约束：`UNIQUE(employee_id, date)` 确保每人每天只有一条记录。

#### verification_codes（验证码表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| email | VARCHAR(100) | 邮箱 |
| code | VARCHAR(5) | 5位数字验证码 |
| expires_at | DATETIME | 过期时间（2分钟有效） |
| used | TINYINT(1) | 是否已使用 |
| created_at | DATETIME | 创建时间 |

## 四、API 接口设计

### 4.1 认证接口

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /api/auth/login | 登录，返回 JWT | 否 |
| POST | /api/auth/register | 注册（需验证码） | 否 |
| POST | /api/auth/reset-password | 重置密码（需验证码+用户名+邮箱匹配） | 否 |
| POST | /api/email/send-code | 发送注册验证码（邮箱 ≤3 账号） | 否 |
| POST | /api/email/send-reset-code | 发送重置密码验证码（用户名+邮箱匹配） | 否 |

### 4.2 业务接口（均需 JWT 认证）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/products | 商品列表 |
| POST | /api/products | 新增商品 |
| PUT | /api/products/:id | 更新商品 |
| DELETE | /api/products/:id | 删除商品（存在关联销售/进货时拒绝） |
| GET | /api/sales?start=&end= | 销售记录（可选日期范围） |
| POST | /api/sales | 新增销售（事务：校验库存→插入记录→扣减库存） |
| GET | /api/purchases?start=&end= | 进货记录（可选日期范围） |
| POST | /api/purchases | 新增进货（事务：插入记录→增加库存） |
| GET | /api/employees | 员工列表 |
| POST | /api/employees | 新增员工 |
| PUT | /api/employees/:id | 更新员工 |
| DELETE | /api/employees/:id | 删除员工（级联删除考勤） |
| GET | /api/attendance?employeeId=&date= | 考勤记录（可选筛选） |
| POST | /api/attendance/clock-in | 上班打卡 |
| POST | /api/attendance/clock-out | 下班签退 |

### 4.3 响应格式

成功响应：`{ "token": "...", "user": {...} }` 或直接返回数据对象/数组

错误响应：`{ "error": "错误描述" }`，HTTP 状态码 400/401/500

销售/进货/打卡 操作额外返回 `{ "ok": true/false, "error": "..." }`

## 五、前端架构

### 5.1 路由设计

| 路径 | 组件 | 说明 | 需要登录 |
|------|------|------|----------|
| /login | Login.vue | 登录页 | 否 |
| /register | Register.vue | 注册页 | 否 |
| /forgot-password | ForgotPassword.vue | 重置密码页 | 否 |
| /dashboard | Dashboard.vue | 仪表盘 | 是 |
| /products | Products.vue | 商品管理 | 是 |
| /sales | Sales.vue | 销售记录 | 是 |
| /purchases | Purchases.vue | 进货记录 | 是 |
| /employees | Employees.vue | 员工管理 | 是 |

路由守卫：未登录访问需认证页面 → 重定向到 `/login`；已登录访问登录/注册 → 重定向到 `/dashboard`

### 5.2 状态管理

每个业务模块对应一个 Pinia Store，Store 通过调用 `src/api/` 层的接口获取数据：

```
Views → Stores → API (Axios) → Vite Proxy → Express → MySQL
```

数据流：Store 暴露 `fetchXxx()` 加载数据，`addXxx()` / `updateXxx()` / `deleteXxx()` 修改数据后自动刷新本地状态。

### 5.3 API 层设计

- `client.js`：Axios 实例，请求拦截器自动附加 JWT，响应拦截器处理 401 自动跳转登录
- 每个业务模块独立 API 文件，处理字段驼峰/下划线互转

## 六、安全设计

1. **密码加密**：bcrypt（cost = 10）哈希存储
2. **JWT 认证**：24小时有效期，前端存储于 localStorage
3. **邮箱验证**：注册/重置密码需5位数字验证码，2分钟有效，60秒冷却
4. **SQL 注入防护**：全部使用参数化查询（mysql2 prepared statements）
5. **商品删除保护**：存在关联销售/进货记录时拒绝删除（FK RESTRICT）
6. **销售库存保护**：使用 MySQL 事务（FOR UPDATE 行锁 → 检查库存 → 扣减）
7. **邮箱绑定限制**：同一邮箱最多绑定 3 个账号
8. **跨域**：Express 配置 CORS，Vite 开发环境使用代理

## 七、部署说明

### 环境要求
- Node.js ≥ 18
- MySQL 8.x

### 启动步骤

```bash
# 1. 安装后端依赖
cd server && npm install

# 2. 初始化数据库（首次运行）
node setup-db.js

# 3. 生成仿真数据
node reseed.js

# 4. 配置邮件（修改 mail-config.js 中的 SMTP 信息）

# 5. 启动后端
node index.js          # 运行在 localhost:3000

# 6. 安装前端依赖
cd .. && npm install

# 7. 启动前端
npm run dev            # 运行在 localhost:5173
```
